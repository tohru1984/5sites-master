## 技術メモ

### 記事データ構造
```typescript
interface Article {
  id: string;
  title: string;
  excerpt: string;
  content: string;
  category: 'reading' | 'psychology' | 'martial-arts';
  tags: string[];
  coverImage: string;
  publishedAt: string;
  readingTime: number;
  author: string;
}
```

### ルーティング
- `/` - ホームページ
- `/articles` - 記事一覧
- `/articles/:id` - 記事詳細

### デプロイフロー
1. コード変更 → `main`ブランチにプッシュ
2. GitHub Actions自動実行
3. ビルド（`pnpm build`）
4. FTPアップロード（Xserver）
5. 本番環境反映（約5-10分）

## 学んだこと

1. **webdev_save_checkpointの重要性**
   - チェックポイント作成時に自動的にGitコミット
   - 手動でのgit pushが必要な場合がある

2. **GitHubリモート設定**
   - webdevプロジェクトはS3リモートを使用
   - GitHubへのプッシュには別途リモート追加が必要
