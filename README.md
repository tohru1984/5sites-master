# 5サイト統合管理リポジトリ

このリポジトリは、ユーザー様の5サイト計画を統合管理するためのMasterリポジトリです。Manusが全スレッドを横断してプロジェクトを管理・運営するための中央ハブとして機能します。

## 目的

- 全5サイトの進捗状況を一元管理
- ユーザー様のプロフィール・好み・指示履歴を記録
- 各サイトの設定情報・収益データを集約
- スレッド間でのコンテキスト共有
- 週次・月次レポートの自動生成

## リポジトリ構成

```
5sites-master/
├── README.md                    # このファイル
├── USER_PROFILE.md              # ユーザープロフィール・好み・指示履歴
├── MASTER_TODO.md               # 全サイト統合TODOリスト
├── WEEKLY_REPORT.md             # 週次進捗レポート
├── MONTHLY_REPORT.md            # 月次収益レポート
├── sites/
│   ├── site1-tatakau-shosai.md  # サイト#1：闘う書斎
│   ├── site2-[TBD].md           # サイト#2（未定）
│   ├── site3-[TBD].md           # サイト#3（未定）
│   ├── site4-[TBD].md           # サイト#4（未定）
│   └── site5-[TBD].md           # サイト#5（未定）
├── ideas/
│   ├── site-themes.md           # サイトテーマ候補
│   ├── article-ideas.md         # 記事アイデアストック
│   └── monetization-strategy.md # 収益化戦略
└── logs/
    └── session-logs/            # セッションごとの作業ログ
```

## 管理方針

### 1. プロジェクト管理
- 各サイトの進捗を`sites/`ディレクトリで個別管理
- 統合TODOリストで優先順位を明確化
- 週次レポートで全体像を把握

### 2. コンテキスト共有
- ユーザープロフィールで好み・指示パターンを学習
- セッションログで過去の会話・決定事項を記録
- 次回スレッドで即座にコンテキスト復元

### 3. 収益最適化
- 各サイトのPV・収益データを集約
- A/Bテスト結果を記録
- 成功パターンを他サイトに展開

## 更新頻度

- **リアルタイム**: ユーザー指示、重要な決定事項
- **日次**: 作業進捗、記事公開
- **週次**: 週次レポート、全サイト進捗
- **月次**: 収益レポート、戦略見直し

## 次回スレッド開始時のテンプレート（ユーザー様向け）

新しいスレッドでManusに作業を依頼する際は、以下のテンプレートをコピペしてください：

### 基本テンプレート
```
【5サイト計画】継続作業をお願いします
Masterリポジトリ: https://github.com/tohru1984/5sites-master
このリポジトリをクローンして、USER_PROFILE.mdとMASTER_TODO.mdを確認してから作業を開始してください。
```

### 優先タスク指定版
```
【5サイト計画】継続作業
Master: https://github.com/tohru1984/5sites-master
優先タスク: サイト#1の記事追加
```

### 特定サイト作業版
```
【5サイト計画】サイト#2の立ち上げ
Master: https://github.com/tohru1984/5sites-master
```

## 使用方法（Manus向け）

### 新規スレッド開始時
1. このリポジトリをクローン
2. `USER_PROFILE.md`でユーザー様のコンテキストを復元
3. `MASTER_TODO.md`で現在の優先タスクを確認
4. 最新の`WEEKLY_REPORT.md`で全体状況を把握

### セッション終了時
1. 作業内容を該当サイトのmdファイルに記録
2. `MASTER_TODO.md`を更新（完了タスクをチェック、新規タスクを追加）
3. セッションログを`logs/session-logs/`に保存
4. GitHubにコミット・プッシュ

### 週次レポート作成時
1. 全サイトの進捗を集計
2. `WEEKLY_REPORT.md`を更新
3. 次週の目標を設定
