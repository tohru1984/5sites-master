# Manus Master リポジトリ

このリポジトリは、ユーザー様のプロジェクト全体を統合管理するためのMasterリポジトリです。Manusが全スレッドを横断してプロジェクトを管理・運営するための中央ハブとして機能します。

## 目的

- 全プロジェクトの進捗状況を一元管理
- ユーザー様のプロフィール・好み・指示履歴を記録
- タスク・収益データ・設定情報を集約
- スレッド間でのコンテキスト共有
- 週次・月次レポートの自動生成

## リポジトリ構成

```
manus-master/
├── README.md                    # このファイル
├── MANUS_KNOWLEDGE.md           # 統合ナレッジベース（最重要）
├── USER_PROFILE.md              # ユーザープロフィール・好み・指示履歴
├── MASTER_TODO.md               # 全プロジェクト統合TODOリスト
├── info-product-summary.md      # 情報商材プロジェクトサマリー
├── sites/
│   ├── site1-tatakau-shosai.md  # サイト#1：闘う書斎
│   ├── site2-[TBD].md           # サイト#2（未定）
│   ├── site3-[TBD].md           # サイト#3（未定）
│   ├── site4-[TBD].md           # サイト#4（未定）
│   └── site5-[TBD].md           # サイト#5（未定）
└── logs/
    └── session-logs/            # セッションごとの作業ログ
```

## 管理方針

### 1. GitHubとNotionの役割分担

| プラットフォーム | 役割 | 主な管理対象 |
|---|---|---|
| **GitHub (manus-master)** | **実行と成果物の管理** | コード、設定ファイル、タスクリスト、作業ログ |
| **Notion (AI Master Database)** | **知識と長期記憶の管理** | AI間の共有ナレッジ、学習内容、抽象的な情報 |

### 2. コンテキスト共有

- `MANUS_KNOWLEDGE.md`で統合ナレッジを管理
- `USER_PROFILE.md`でユーザー様の好み・指示パターンを学習
- セッションログで過去の会話・決定事項を記録
- 次回スレッドで即座にコンテキスト復元

### 3. 更新タイミング

- **リアルタイム**: ユーザー指示、重要な決定事項
- **日次**: 作業進捗、記事公開
- **週次**: 週次レポート、全プロジェクト進捗
- **セッション終了時**: タスクリスト、ナレッジベース、Notionへの集約

## 次回スレッド開始時のテンプレート（ユーザー様向け）

新しいスレッドでManusに作業を依頼する際は、以下のテンプレートをコピペしてください：

### 基本テンプレート
```
継続作業をお願いします
Masterリポジトリ: https://github.com/tohru1984/manus-master
このリポジトリをクローンして、MANUS_KNOWLEDGE.mdとMASTER_TODO.mdを確認してから作業を開始してください。
```

### 優先タスク指定版
```
継続作業
Master: https://github.com/tohru1984/manus-master
優先タスク: [具体的なタスク内容]
```

## 使用方法（Manus向け）

### 新規スレッド開始時
1. このリポジトリをクローン: `gh repo clone tohru1984/manus-master`
2. `MANUS_KNOWLEDGE.md`でコンテキストを復元
3. `MASTER_TODO.md`で現在の優先タスクを確認
4. 必要に応じて`USER_PROFILE.md`で詳細情報を確認

### セッション終了時
1. 作業内容を該当ファイルに記録
2. `MASTER_TODO.md`を更新（完了タスクをチェック、新規タスクを追加）
3. `MANUS_KNOWLEDGE.md`を更新（新しい学習事項があれば）
4. セッションログを`logs/session-logs/`に保存
5. GitHubにコミット・プッシュ
6. Notionの`manus.md`にも重要な更新を反映
